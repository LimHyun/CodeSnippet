Aptana 3를 사용해서 github 셋팅중 알게 된 사실 정리
=====================================================

*이 내용은 추후 블로그나 위키쪽으로 이동할 예정임


1. ssh접속을 위한 인증키 생성
github접속을 위해 ssh를 사용해야 하는데, 이를 위해서는 ssh용 키를 생성해야 한다.
이 부분은 github에 있는 공식 페이지를 참고해서 작업하니 특별히 문제는 없었는데,
작업을 하는 콘솔을 착각하는 바람에 같은 작업을 두번했음.
가급적이면 aptana에 있는 콘솔을 사용하거나, git bash를 사용하는게 좋은듯.


2. ssh인증키 설정
인증키 생성후에 샘플을 만들어서 push를 하려고 하니 접근에러가 발생했다.
기본적으로 콘솔상에서는 문제가 없었는데, 컨텍스트 메뉴에 있는 pull을 사용하면 접근제한이 걸리는 상황이 되버렸다.
이를 해결하기 위해 aptana의 콘솔을 사용도 해보고 git bash도 사용해 봤는데,
결론은 aptana의 환경설정에 있는 ssh경로에 인증서 파일들을 위치시켜야 한다는 것이었다.
http://web-couch.com/2011/12/26/aptana-3-fur-git-und-github-einrichten/ (독일어 사이트)

3. push 시 이름이 unknown 으로 나오는 문제
몇번 push를 하면서 웹으로 확인해 보니, 이름이 unknown 으로 나오고 있어서 이를 해결하기 위해 다시 검색.
결론은 aptana 의 셋팅(환경설정에 있는 유저정보 / ssh 인증서에 등록된 정보) 와 별도로,
콘솔상에서 유저정보를 셋팅해 줘야 한다는 것이었다.
아마도 프로젝트별로 이름이 달라질 수 있다는 것을 염두에 둔것인지도 모르겠다.
http://stackoverflow.com/questions/15805770/setup-git-user-in-aptana
